# Estratégia de Avaliação

## Distribuição de pontos

- Unidade 1: 30 pontos
- Unidade 2: 30 pontos
- Unidade 3: 40 pontos
- Total: 100 pontos

## Composição sugerida

### Unidade 1 (30)

- Práticas iniciais de HTTP, NestJS e validação
- Checkpoint de API CRUD sem autenticação
- Avaliação prática da unidade

### Unidade 2 (30)

- Práticas de formulários, autenticação e sessões
- Checkpoint de integração com API externa
- Avaliação prática da unidade

### Unidade 3 (40)

- Práticas de banco relacional e NoSQL
- Projeto final com deploy
- Apresentação técnica
- Qualidade técnica do repositório

## Critérios gerais

1. Funcionamento correto da API e aderência aos requisitos.
2. Organização do código em módulos, serviços e controladores.
3. Validação de entradas, tratamento de erros e autenticação.
4. Uso adequado de persistência e integração com serviços externos.
5. Documentação, testes e capacidade de publicação da aplicação.
