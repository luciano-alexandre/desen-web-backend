# Projeto Integrador

## Proposta

Desenvolver uma aplicação Web back-end em `NestJS` para um cenário realista de
TSI, como catálogo de serviços, sistema de reservas, gestão de conteúdos,
controle de atendimentos ou outra solução aprovada pelo professor.

## Requisitos mínimos

- API REST organizada por módulos.
- Validação de dados com DTOs.
- Autenticação de usuários.
- Ao menos um fluxo com banco relacional.
- Ao menos um fluxo com banco não relacional ou integração externa relevante.
- Documentação de rotas.
- Deploy funcional em ambiente público.

## Entregas

1. Proposta de escopo.
2. Estrutura inicial e backlog técnico.
3. Entrega parcial funcional.
4. Release candidate com deploy.
5. Apresentação final.

## Critérios de avaliação

1. Correção funcional.
2. Qualidade da arquitetura.
3. Segurança mínima e validação.
4. Persistência e integração.
5. Deploy e documentação.
