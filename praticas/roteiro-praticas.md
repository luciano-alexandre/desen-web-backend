# Roteiro de Práticas

## Princípios

- Cada prática deve produzir evidência em repositório.
- O código deve ser incremental e reaproveitado nas unidades seguintes.
- Sempre incluir README curto com instruções de execução local.

## Sequência sugerida

1. API inicial e rotas.
2. DTOs e validação.
3. Tratamento de erros.
4. Testes básicos.
5. Sessão e autenticação.
6. Integração com API externa.
7. PostgreSQL.
8. MongoDB.
9. Docker e deploy.
